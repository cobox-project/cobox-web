# Cobox プロダクト仕様書

## 1. 概要

Coboxは、複数チャネル（Email / LINE / Instagram / Facebook）からの顧客メッセージを一元管理するカスタマーコミュニケーションツールです。

---

## 2. メッセージステータス定義

### 2.1 ステータス一覧

| ステータス | 内部値 | 意味 |
|-----------|--------|------|
| **新着** | `new` | まだ誰も対応していない。アクション待ち |
| **対応中** | `in_progress` | 誰かが対応を開始した（アサインまたは返信） |
| **完了** | `completed` | 対応が終了した |
| **対応なし** | `no_action` | 対応不要と判断した |

### 2.2 ステータス遷移ルール

#### 遷移図

```
                    ┌──────────────────────────────────┐
                    │            新着                   │
                    │  (初期状態: メッセージ受信時)       │
                    └──┬────────┬───────────────────────┘
                       │        │
            自分/他人を  │        │ スタッフが返信した
            アサイン    │        │
                       ▼        ▼
                    ┌──────────────────────────────────┐
                    │           対応中                   │
                    └──┬────────┬───────────────────────┘
                       │        │
            完了ボタン  │        │ 対応なしボタン
                       ▼        ▼
              ┌──────────┐  ┌──────────┐
              │   完了    │  │ 対応なし  │
              └────┬─────┘  └────┬─────┘
                   │             │
                   │  顧客から   │  顧客から
                   │  再返信     │  再返信
                   ▼             ▼
              ┌──────────────────────────────────┐
              │          新着（再浮上）            │
              └──────────────────────────────────┘
```

#### 全イベント×現在ステータスの遷移表

| イベント | 新着 → | 対応中 → | 完了 → | 対応なし → |
|---------|--------|----------|--------|-----------|
| **誰かをアサイン** | 対応中 | 対応中（変化なし） | 変化なし | 変化なし |
| **アサインを全員外す** | — | **対応中のまま** | — | — |
| **スタッフが返信** | 対応中 | 対応中（変化なし） | 対応中 | 対応中 |
| **完了ボタン押下** | 完了 | 完了 | 対応中に戻す（トグル） | 完了 |
| **対応なしボタン押下** | 対応なし | 対応なし | 対応なし | 対応中に戻す（トグル） |
| **顧客から新規メッセージ受信** | 新着（変化なし） | 対応中（変化なし） | **新着に再浮上** | **新着に再浮上** |
| **完全に削除** | 削除 | 削除 | 削除 | 削除 |

#### 重要ルール

1. **アサインを外しても対応中のまま** — 一度「対応中」になったら、アサインを外しただけでは「新着」に戻らない。引き継ぎ時に宙に浮かない。
2. **返信 = 対応開始とみなす** — アサインしなくても返信した時点で「対応中」に移行。
3. **完了/対応なしからの再浮上** — 顧客から再返信が来たら自動的に「新着」に戻す。見落とし防止。アサイン情報は保持する。
4. **完了/対応なしのトグル** — 完了状態で完了を再押下 → 対応中に戻す。対応なしも同様。
5. **スタッフが返信した場合の遷移** — 完了/対応なしから直接「対応中」に移行（スタッフの能動的なアクション）。

### 2.3 フォルダ分類条件

| フォルダ | 表示条件 |
|---------|---------|
| 新着 | `status === "new"` |
| 対応中 | `status === "in_progress"` |
| 完了 | `status === "completed"` |
| 対応なし | `status === "no_action"` |
| 自分が担当 | `assignees` に自分が含まれる（ステータス問わず） |
| メンションされた | チーム内メモで `@自分` が含まれるスレッド |
| お気に入り | `isFavorite === true` |

> **注意**: フォルダ分類は `status` フィールドのみで判定する。アサインの有無では判定しない。

## 3. メッセージID（会話スレッド）のルール

### 3.1 チャネルごとのスレッド生成条件

**共通条件**: すべてのチャネルにおいて、同一スレッドとみなすには「ステータスが新着または対応中である」（完了・対応なしではない）ことが前提条件となる。ステータスが完了または対応なしの場合は、下記条件を満たしていてもスレッド生成条件を超えたものとして新規メッセージIDが発行される。

| チャネル | 同一スレッドとみなす条件 | 新規メッセージIDになる条件 |
|---------|----------------------|------------------------|
| **Email** | 同一メールスレッド（`In-Reply-To` / `References` ヘッダが一致）かつ、前回メッセージから **7日以内** | `In-Reply-To` / `References` ヘッダが一致しない、または前回メッセージから **7日以上** 経過した新規メッセージ |
| **LINE** | 同一ユーザーからの連続メッセージで、前回メッセージから **7日以内** | 前回メッセージから **7日以上** 経過した新規メッセージ |
| **Instagram** | 同一ユーザーからの連続メッセージで、前回メッセージから **7日以内** | 前回メッセージから **7日以上** 経過した新規メッセージ |
| **Facebook** | 同一ユーザーからの連続メッセージで、前回メッセージから **7日以内** | 前回メッセージから **7日以上** 経過した新規メッセージ |

### 3.2 メッセージIDフォーマット

- 形式: `#00001` から連番（ゼロ埋め5桁）
- スコープ: ワークスペース全体でユニーク
- 表示位置: メッセージスレッド下部

### 3.3 再浮上時の扱い

- 完了/対応なしのスレッドに顧客から新規メッセージが来た場合:
  - **同一スレッド内に追加**される（上記のスレッド生成条件を満たす場合）
  - ステータスが「新着」に戻る
  - アサイン情報は保持される
  - スレッド生成条件を超えている場合は**新規メッセージID**が発行される

## 4. メッセージ紐づけ

### 4.1 概要

同一連絡先に紐づく異なるメッセージスレッド同士を手動で紐づけることができる。

### 4.2 紐づけルール

- 紐づけは双方向（A→BリンクしたらB→Aもリンク）
- 同一連絡先のスレッドのみ紐づけ可能
- 紐づけ数の上限なし
- 解除も双方向で行われる

### 4.3 紐づけメッセージの表示

#### 表示位置
- 紐づけ先が**過去メッセージ**（現在のスレッドより前の日時） → スレッド上部に表示
- 紐づけ先が**未来メッセージ**（現在のスレッドより後の日時） → スレッド下部に表示（ただし入力フォームより上）

#### 表示内容
| チャネル | 表示するテキスト |
|---------|----------------|
| Email | 件名 (`subject`) |
| Instagram | 最初のメッセージ本文 |
| LINE | 最初のメッセージ本文 |
| Facebook | 最初のメッセージ本文 |

※ メッセージ履歴のタイトル表示と同一形式を使用する。

## 5. アサイン

### 5.1 アサインの種類

- **自分をアサイン**: 現在のログインユーザーを担当者に追加
- **他メンバーをアサイン**: ドロップダウンから選択
- **アサイン解除**: 担当者を外す

### 5.2 自動アサイン

- 返信テキストボックスに1文字以上入力した時点で、未アサインの場合は自動的に自分をアサインする

### 5.3 アサイン表示

| 状態 | ヘッダー表示 |
|------|------------|
| 未アサイン | 「未アサイン｜自分にアサイン」連結ボタン |
| 自分がアサイン済 | 担当者名表示 + ドロップダウン |
| 他メンバーがアサイン済 | 「自分をアサイン追加」+ ドロップダウン |

## 6. 連絡先

### 6.1 連絡先フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| 氏名 | ○（新規作成時） | 表示名 |
| フリガナ | — | ソート用 |
| 会社名 | — | |
| 会社名フリガナ | — | |
| 電話番号 | — | |
| メールアドレス | — | 自動連携の場合は編集不可 |
| SNSチャネル | — | Instagram / LINE / Facebook ハンドル |
| グループ | — | 複数選択可 |
| メモ | — | 自由入力 |

### 6.2 自動連携

チャネル接続時に受信したメッセージの送信者情報から、自動的に連絡先を作成・紐づけする。
自動連携された連絡先のチャネル情報は編集不可（`isAutoLinked: true`）。

## 7. 一括送信

### 7.1 送信対象

- **グループ指定**: 1つ以上のグループを選択
- **個別指定**: 連絡先を個別に選択

### 7.2 送信チャネル

- Email のみ（一括送信はメールアカウント経由）

### 7.3 差し込み変数

| 変数 | 展開内容 |
|------|---------|
| `{{名前}}` | 連絡先の氏名（フルネーム） |
| `{{会社名}}` | 連絡先の会社名 |

> 管理画面の「テンプレート > 連絡先・定型文」から連絡先変数（編集不可）と定型文（カスタム追加可能）を管理できる。

### 7.4 下書き

- 自動保存なし（明示的に保存ボタンで保存）
- 下書き一覧は送信ボタン左にボタン表示（下書きが存在する場合のみ）
- クリックでモーダル表示 → 選択で編集再開

### 7.5 未定義変数の警告

- 本文中に `{{xxx}}` 形式の文字列があり、定義済みの連絡先変数・定型文キーに一致しない場合、警告を表示する
- 警告は情報提供のみで、送信をブロックしない

## 8. テンプレート

### 8.1 テンプレート管理

- 管理タブ（設定）内に「メッセージテンプレート」メニューを追加
- 操作: 一覧表示 / 追加 / 編集 / 削除 / 並び替え

### 8.2 テンプレート構造

| フィールド | 説明 |
|-----------|------|
| テンプレート名 | 管理用の名称 |
| 件名 | メール件名（任意） |
| 本文 | メッセージ本文（差し込み変数使用可） |

### 8.3 テンプレート利用

- メッセージ作成画面でテンプレートボタンから選択・適用
- テンプレート選択時に件名・本文を上書き（確認なし）

### 8.4 連絡先変数と定型文

管理画面の「テンプレート > 連絡先・定型文」タブで管理する。

| 種別 | 説明 |
|------|------|
| **連絡先** | 連絡先情報から自動展開される変数（`{{名前}}`、`{{会社名}}`）。編集不可。 |
| **定型文** | ユーザーが自由に追加・編集・削除できるカスタムキー。`{{キー名}}` 形式で本文に挿入可能。 |

| プロ (¥2,980/月) | 無制限 | 無料枠 1,000通/月、超過 3円/通 |

## 11. 入力ロック（タイピングロック）

### 11.1 概要

同一会話スレッドに対して、あるスタッフが返信を入力中の場合、他のスタッフの返信入力をロックし、重複返信を防止する。

### 11.2 ロック開始条件

- スタッフが返信テキストボックスに **1文字以上** 入力した時点で、ほぼリアルタイムにサーバーへタイピング開始イベントを送信する
- サーバーは該当会話に `typingUser` を記録し、他の接続クライアントにほぼリアルタイムに通知する

### 11.3 ロック解除条件

以下のいずれかで解除される:

| 条件 | 説明 |
|------|------|
| メッセージ送信 | 返信を送信した時点で即座に解除 |
| テキスト全削除 | 入力欄が空になった時点で解除 |
| 会話を離れる | 別の会話を選択した時点で解除 |
| タイムアウト | 最後のキー入力から **30秒** 経過で自動解除 |
| 切断 | 接続切断時にサーバー側で自動解除 |

### 11.4 ロック中の他スタッフの表示

- 返信テキストボックスを **非活性化**（`disabled`）し、半透明のオーバーレイを表示する
- オーバーレイには入力中のスタッフ名をリアルタイム表示する: `「{スタッフ名} が入力中...」`
- チーム内メモ欄はロック対象外（複数人が同時にメモ可能）

### 11.5 エッジケース

- 同一ユーザーが複数タブで開いている場合、最初にタイピングを開始したタブが優先される
- ロック中のスタッフが30秒以上操作しない場合、タイムアウトで自動解除し、他のスタッフが入力可能になる
- ネットワーク不安定時は、サーバー側で接続切断を検知し、ロックを自動解除する
